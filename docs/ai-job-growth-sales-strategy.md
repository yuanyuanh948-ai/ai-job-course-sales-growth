# AI就业班增长与社群转化策略归档

更新日期：2026-06-15

## 背景

当前业务有两类可触达人群：

1. 原有 `AI+科研` 产品学生，信任度高，已验证可二转。
2. 公域进入的传统行业 / 非存量用户，信任低，需求散，需要先教育再转化。

这两类人不能用同一套话术和同一条漏斗处理。

## 关键判断

### 1. 老学员是现金流线

这部分人已经被产品验证过，适合做二转和复购。

销售重点不是建立信任，而是把他们已有的科研、课程、项目经历翻译成就业资产。

建议承接路径：

- 就业风险诊断
- 科研经历转就业表达公开课
- 7天项目包装任务
- 就业班二转

### 2. 公域是养熟线

这部分人信任弱、基础不一、诉求更模糊，不能直接硬推就业班。

更适合先进入社群，用小产物建立认知和信任，再分流到不同产品。

建议承接路径：

- 诊断表
- 5天AI工作竞争力诊断营
- 小作品任务
- 一对一诊断

### 3. 社群的职责是筛选，不是灌输

社群不应该承担全部成交压力。

它的任务只有三个：

- 让用户看到 AI 和自身工作/求职问题的关系
- 让团队判断是否适合就业班
- 让高意向用户主动浮出水面

## 推荐结构

### 老学员漏斗

私域触达 -> 就业风险诊断 -> 转就业公开课 -> 项目包装任务 -> 就业班报名

### 公域漏斗

小红书 / 淘宝 -> 诊断表 -> 社群体验营 -> 小作品点评 -> 分流到就业班 / 入门班 / 长期养熟

## 公域社群设计

建议把社群设计为 `5天AI工作竞争力诊断营`，而不是资料群或单场公开课群。

整体设计原则：**投入成本递增，获得感也递增，每一步给用户模板降低门槛。**

### 第 0 天：入群筛选 + 破冰

- 填写 AI工作 / 就业竞争力诊断表。
  - 核心字段：行业、岗位、学历、编程基础、当前最想解决的问题、是否求职或转岗、每周可投入时间。
- 入群自我介绍接龙：**行业 + 岗位 + 一个你最想问AI的问题**。降低用户沉默率，让团队提前感知人群画像。
- 晚间发送第 1 天预告海报/文案，制造期待感。

**目标**：用户进群后有事做，不只是填表就等着。

### 第 1 天：认知建立 + 痛点挖掘

- 主题：AI不是学工具，而是重做工作流。
- 任务：提交一份 **"我的AI工作痛点清单"**（结构化模板）。
  - 模板字段：行业 → 岗位 → 当前最卡壳的1个问题 → 希望AI解决什么 → 自己尝试过什么
  - 给示例参考，避免用户不知道写多深。
- 午间互动：群内接龙"晒出你的痛点"，互相点赞评论，营造社群氛围。
- 晚间：发第 2 天预告 + 2 个行业场景示例（覆盖不同人群类型）。

**目标**：让用户写出足够具体的痛点，为后续场景拆解和小作品铺垫。

### 第 2 天：行业场景拆解

- 主题：你的行业里，AI 能先落在哪些地方。
- 任务：提交一张 **AI应用场景卡片**（提供标准模板和示例）。
  - 卡片模板格式：行业 → 岗位 → 场景描述 → AI介入点 → 预期效果
  - 提供 2-3 个不同行业的完整示例（技术类、运营类、传统行业类），用户填空式完成。
- 晚间互动：**互看互评**——每人看 2 份其他人的场景卡片，写一句反馈。

**目标**：用户不用从白纸开始，填空式参与降低焦虑；互评增加社交粘性。

### 第 3 天：小作品搭建（阶梯门槛）

- 主题：做一个你行业的 **最小AI工作流**。
- 提供三级阶梯选项，用户根据自己能力选一级：
  - **选项 A（无代码）**：用 AI 工具（ChatGPT / Claude / Kimi）完成一个工作流设计稿，输出为文字流程描述（适合 0 基础）。
  - **选项 B（半成品填空）**：基于给定的 Prompt 模板或流程图，填入自己的行业数据/场景，产出定制化方案（适合有概念但缺执行的用户）。
  - **选项 C（轻量实操）**：用 AI 辅助处理自己的一份真实数据或文档，产出分析结果（适合有一定动手能力的用户）。
- 提交窗口延长至 **第 4 天中午**（实际 1.5 天），给用户充足时间。

**目标**：0 基础也能做出东西，不再被门槛卡死；有基础的可以做更深，自然分层。

### 第 4 天：点评 + 加餐（全程有内容）

- 上午：作业截止 + 加餐内容 **"AI求职/工作效率提效 Top 10 工具清单"**（让没交作业的人也有收获）。
- 下午：公开点评 15-20 份作业，覆盖更多学员；设 **优秀作业展示墙**，给予群体认可。
- 持续开放一对一诊断预约通道，不做统一时间截止。
- 晚间：发第 5 天路线说明会预告。

**目标**：没被点评到的人也有内容可看，全程保持参与感。

### 第 5 天：路线说明会 + 结营转化

- 讲清三条路径：工具增强线、行业AI应用线、AI就业技术线。
- 每条路径配一个 **简短的真实学员案例**（脱敏），让听众看到"和我类似的人怎么选的"。
- 结营时给所有提交过作业的用户发一份 **"AI工作竞争力诊断报告"**（自动生成或基于诊断表填写的简单模板），作为离营资产，增加分享传播可能。
- 结营不散群：转为 **"AI就业资讯群"**（节奏放缓，每周 1-2 次内容推送），持续养熟低意向用户。
- 承接一对一诊断成交：
  - A 类（有基础、有时间、有求职或转岗需求）：推就业班。
  - B 类（行业场景明确但技术弱）：推低门槛训练营或继续培育。
  - C 类（需求弱、行动弱）：进入长期私域养熟，不强追。

**目标**：结营不是结束，而是进入下一个池子的入口。

## 销售分层

### A 类：就业班高潜

特征：

- 有求职或转岗时间点
- 有学习时间
- 能接受写代码或补技术
- 对项目作品集感兴趣

销售方向：就业班。

### B 类：行业AI应用潜力

特征：

- 行业经验明确
- 编程弱
- 更关心工作提效和原岗位竞争力

销售方向：低价体验营、行业AI工作流课，后续再升级。

### C 类：长期养熟

特征：

- 只看不动
- 需求不清
- 不愿投入

销售方向：朋友圈、公众号、长期社群维护，不强追。

## 本月优先级

1. 复盘已成交的 70 人，提炼成交画像、理由和拒绝点。
2. 设计老学员专属二转活动。
3. 设计公域 5 天游程和诊断表。
4. 每周从第一期班提炼可复用案例。
5. 建立转化看板，拆清每一层漏斗数据。

## 指标看板

- 入群人数
- 诊断表提交率
- 第 1 天任务提交率
- 小作品提交率
- 公开课到场率
- 一对一诊断预约率
- A/B/C 分层比例
- A 类成交率
- B 类进入下一产品率

## 表达边界

不能承诺包就业、薪资、offer、转岗成功。

可以承诺可控交付：学习路径、项目训练、作业点评、简历表达、面试讲解准备、作品集产出。
