# AI就业班增长与社群转化策略归档

更新日期：2026-06-15

## 背景

当前业务有两类可触达人群：

1. 原有 `AI+科研` 产品学生，信任度高，已验证可二转。
2. 公域进入的传统行业 / 非存量用户，信任低，需求散，需要先教育再转化。

这两类人不能用同一套话术和同一条漏斗处理。

## 关键判断

### 1. 老学员是现金流线

这部分人已经被产品验证过，适合做二转和复购。

销售重点不是建立信任，而是把他们已有的科研、课程、项目经历翻译成就业资产。

建议承接路径：

- 就业风险诊断
- 科研经历转就业表达公开课
- 7天项目包装任务
- 就业班二转

### 2. 公域是养熟线

这部分人信任弱、基础不一、诉求更模糊，不能直接硬推就业班。

更适合先进入社群，用小产物建立认知和信任，再分流到不同产品。

建议承接路径：

- 诊断表
- 5天AI工作竞争力诊断营
- 小作品任务
- 一对一诊断

### 3. 社群的职责是筛选，不是灌输

社群不应该承担全部成交压力。

它的任务只有三个：

- 让用户看到 AI 和自身工作/求职问题的关系
- 让团队判断是否适合就业班
- 让高意向用户主动浮出水面

## 推荐结构

### 老学员漏斗

私域触达 -> 就业风险诊断 -> 转就业公开课 -> 项目包装任务 -> 就业班报名

### 公域漏斗

小红书 / 淘宝 -> 诊断表 -> 社群体验营 -> 小作品点评 -> 分流到就业班 / 入门班 / 长期养熟

## 公域社群设计

建议把社群设计为 `5天AI工作竞争力诊断营`，而不是资料群或单场公开课群。

第 0 天：入群筛选

- 填写 AI工作 / 就业竞争力诊断表。
- 核心字段：行业、岗位、学历、编程基础、当前最想解决的问题、是否求职或转岗、每周可投入时间。

第 1 天：建立认知

- 主题：AI不是学工具，而是重做工作流。
- 任务：写出当前工作或求职中最想被 AI 解决的一个问题。

第 2 天：行业场景拆解

- 主题：你的行业里，AI 能先落在哪些地方。
- 任务：提交一张 `AI应用场景卡片`。

第 3 天：小作品搭建

- 主题：做出一个最小 AI 工作流。
- 产物可选：AI简历优化流程、行业资料整理流程、报告生成流程、客户问答知识库雏形、岗位JD分析表。

第 4 天：点评与分层

- 公开点评 5-10 个作业。
- 分为 A/B/C 三类线索。

第 5 天：路线说明会

- 讲清三条路径：工具增强线、行业AI应用线、AI就业技术线。
- 承接一对一诊断，不在群里硬推大课。

第 6-7 天：一对一诊断成交

- A 类：有基础、有时间、有求职或转岗需求，推就业班。
- B 类：行业场景明确但技术弱，推低门槛训练营或继续培育。
- C 类：需求弱、行动弱，进入长期私域养熟。

## 销售分层

### A 类：就业班高潜

特征：

- 有求职或转岗时间点
- 有学习时间
- 能接受写代码或补技术
- 对项目作品集感兴趣

销售方向：就业班。

### B 类：行业AI应用潜力

特征：

- 行业经验明确
- 编程弱
- 更关心工作提效和原岗位竞争力

销售方向：低价体验营、行业AI工作流课，后续再升级。

### C 类：长期养熟

特征：

- 只看不动
- 需求不清
- 不愿投入

销售方向：朋友圈、公众号、长期社群维护，不强追。

## 本月优先级

1. 复盘已成交的 70 人，提炼成交画像、理由和拒绝点。
2. 设计老学员专属二转活动。
3. 设计公域 5 天游程和诊断表。
4. 每周从第一期班提炼可复用案例。
5. 建立转化看板，拆清每一层漏斗数据。

## 指标看板

- 入群人数
- 诊断表提交率
- 第 1 天任务提交率
- 小作品提交率
- 公开课到场率
- 一对一诊断预约率
- A/B/C 分层比例
- A 类成交率
- B 类进入下一产品率

## 表达边界

不能承诺包就业、薪资、offer、转岗成功。

可以承诺可控交付：学习路径、项目训练、作业点评、简历表达、面试讲解准备、作品集产出。
